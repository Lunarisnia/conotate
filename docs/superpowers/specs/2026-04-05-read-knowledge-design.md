# read-knowledge Skill — Design Spec

**Date:** 2026-04-05
**Status:** Approved

---

## Overview

A Claude Code skill that loads one or more existing knowledge base files (generated by `explore-domain`) into the conversation context. Resolves free-form domain input against available knowledge files, extracts the relevant blocks via a bundled Python script, and acknowledges loading with a brief message.

---

## Invocation

- **Skill name:** `read-knowledge`
- **Mode:** Interactive — the skill asks the user which domain(s) to load
- **Example prompt:** *"Which domain(s) do you want to load?"* → user replies `auth`, `src/auth`, or `auth and payments`

---

## Domain Resolution

1. Parse user input into one or more domain candidates (comma-separated, "and"-separated, or space-separated)
2. Glob `docs/conotate/*.md` to get all available slugs
3. For each candidate, fuzzy-match against the slug list (e.g. `auth` → `src-auth`, `payments` → `src-payments`)
4. **Exact / unambiguous match** → proceed
5. **Ambiguous match** (multiple candidates) → present a numbered list and ask the user to pick
6. **No match** → inform the user no knowledge exists for that domain and recommend running `explore-domain`

---

## Extraction & Loading

1. For each resolved slug, run `fetch_domain.py` via `Bash` with the slug and the path to `docs/conotate/<slug>.md`
2. The script extracts the block between `<!-- domain: X -->` and `<!-- end:X -->` and prints it to stdout; exits non-zero with a clear error if the block is not found
3. For multiple domains, run the script per domain and concatenate outputs in order
4. After loading, acknowledge with a brief message:

> "Knowledge base for `src/auth` loaded. Ready to proceed."

> "Knowledge base for `src/auth`, `payments` loaded. Ready to proceed."

This acknowledgement is on by default. The user can ask to skip it.

---

## Skill Structure

```
skills/read-knowledge/
  SKILL.md                    # Skill instructions
  scripts/
    fetch_domain.py           # Extracts delimited domain block from a knowledge file
```

### Allowed tools (frontmatter)

```yaml
allowed-tools:
  - Read
  - Glob
  - Bash(python3 *)
```

### `fetch_domain.py`

- Arguments: `<slug>` `<file_path>`
- Reads the file, finds the block between `<!-- domain: <slug> -->` and `<!-- end:<slug> -->`
- Prints the block to stdout
- Exits non-zero with a descriptive error message if the block is not found

---

## Relationship to explore-domain

`read-knowledge` is the companion skill to `explore-domain`. `explore-domain` generates knowledge files; `read-knowledge` loads them. The file format (delimited blocks, domain-prefixed headings) is defined by `explore-domain` and consumed here without modification.
