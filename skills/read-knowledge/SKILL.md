# read-knowledge

Loads one or more domain knowledge base files (generated by `explore-domain`) into context.

## Invocation

Ask the user:

> "Which domain(s) do you want to load?"

Parse the response into one or more domain candidates. Accept comma-separated, "and"-separated, or space-separated input (e.g. `auth`, `src/auth`, `auth and payments`, `auth, payments`).

## Domain Resolution

1. If `docs/conotate/domains.txt` exists: parse each line as `<description>:<slug>` into a list of `(description, slug)` pairs
2. Glob `docs/conotate/*.md` → derive slugs by stripping prefix and `.md` suffix → add any slug not already in the list as `("", slug)` (backward compatibility for files generated before domains.txt existed)
3. For each candidate (normalize: lowercase, `/` → `-`):
   - Match against slug (case-insensitive substring)
   - Match against description (case-insensitive substring, if non-empty)
   - e.g. `auth` matches `User authentication and session management:src-auth` via description; `src-auth` matches via slug
4. **Unambiguous match** (exactly one slug matches) → proceed with that slug
5. **Ambiguous** (multiple slugs match one candidate) → present a numbered list and ask the user to pick one
6. **No match** → inform the user and stop:
   > "No knowledge file found for `<candidate>`. Run `explore-domain` to generate one."

## Loading

For each resolved `<slug>`:

```bash
python3 ${CLAUDE_SKILL_DIR}/scripts/fetch_domain.py <slug> docs/conotate/<slug>.md
```

Read the output into context silently.

For multiple domains, run the script once per slug and concatenate the outputs in the order the user specified.

## Acknowledgement

After all domains are loaded, print:

> "Knowledge base for `<domain>` loaded. Ready to proceed."

For multiple domains:

> "Knowledge base for `<domain1>`, `<domain2>` loaded. Ready to proceed."

Skip the acknowledgement only if the user explicitly asks to load silently.
